## 컴파일 에러 및 수정

### 에러 1: BooleanToVisibilityConverter.Default 멤버 없음
- **에러 메시지**: `'BooleanToVisibilityConverter' 형식에서 정적 멤버 'Default'을(를) 찾을 수 없습니다.`
- **파일**: `TastyApe73.Wpf.UI/Themes/TastyApe73.xaml:43`
- **원인**: WPF의 기본 `BooleanToVisibilityConverter`에는 `Default` 정적 멤버가 없음
- **수정 방법**:
  1. 커스텀 `BoolToVisibilityConverter` 클래스 생성
  2. XAML에서 StaticResource로 참조하도록 변경
  ```xml
  <!-- Before -->
  Visibility="{TemplateBinding ShowConfetti, Converter={x:Static BooleanToVisibilityConverter.Default}}"

  <!-- After -->
  Visibility="{Binding ShowConfetti, RelativeSource={RelativeSource TemplatedParent}, Converter={StaticResource BoolToVisibilityConverter}}"
  ```

### 에러 2: StackPanel에 Spacing 속성 없음
- **에러 메시지**: `XML 네임스페이스 'http://schemas.microsoft.com/winfx/2006/xaml/presentation'에 'Spacing' 속성이 없습니다.`
- **파일**: `TastyApe73.Wpf.Gallery/MainWindow.xaml:14`
- **원인**: WPF의 `StackPanel`에는 `Spacing` 속성이 없음 (AvaloniaUI에만 존재)
- **수정 방법**: 각 자식 요소에 `Margin` 속성으로 간격 지정
  ```xml
  <!-- Before -->
  <StackPanel Spacing="30">
      <controls:TastyApe73/>

  <!-- After -->
  <StackPanel>
      <controls:TastyApe73 Margin="0,0,0,30"/>
  ```

## 잠재적 런타임 오류

### 1. 컨페티 애니메이션 Canvas.Top
- **위치**: `TastyApe73.xaml` - ConfettiCanvas 애니메이션
- **내용**: `Canvas.Top` attached property에 애니메이션을 적용했으나, Canvas가 Grid 내부에 배치되어 있어 Canvas.Top이 예상대로 동작하지 않을 수 있음
- **확인 필요**: 호버 시 컨페티가 아래로 떨어지는 애니메이션이 정상 동작하는지 확인

### 2. 클리핑 영역 동적 계산
- **위치**: `TastyApe73.xaml` - RootBorder.Clip
- **내용**: MultiBinding을 사용하여 ActualWidth/ActualHeight 기반으로 클리핑 영역을 계산하므로, 컨트롤 크기가 변경될 때 클리핑이 제대로 업데이트되는지 확인 필요

### 3. 스케일 애니메이션 시작값
- **위치**: `TastyApe73.xaml` - ScaleTransform
- **내용**: ScaleX/Y가 초기값 0으로 설정되어 있어 Loaded 이벤트 전까지 컨트롤이 보이지 않음. 이는 의도된 동작이지만, 애니메이션이 트리거되지 않으면 컨트롤이 표시되지 않을 수 있음
