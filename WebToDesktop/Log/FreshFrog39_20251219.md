## 컴파일 에러 및 수정 내용

### 에러 1: XML 주석 내 '--' 문자 불허
- **파일**: `FreshFrog39.Wpf.UI/Themes/FreshFrog39.xaml`
- **라인**: 22
- **에러 메시지**: `MC3000: 'An XML comment cannot contain '--', and '-' cannot be the last character.'`
- **원인**: CSS 변수 `var(--gap)`을 주석으로 문서화할 때, XML 주석 내에서 `--` 문자열이 허용되지 않음
- **수정 방법**: `var(--gap)` → `var([gap])`으로 변경하여 CSS 변수 이름을 대괄호 형식으로 문서화

```xml
<!-- 수정 전 -->
<!-- CSS: display: flex; justify-content: center; align-items: center; gap: var(--gap); -->

<!-- 수정 후 -->
<!-- CSS: display: flex; justify-content: center; align-items: center; gap: var([gap]); -->
```

## Runtime Error 가능성 (직접 확인 필요)

### 1. TemplateBinding과 Converter 조합
- **위치**: `FreshFrog39.xaml` 라인 43, 52
- **내용**: `Margin="{TemplateBinding DotGap, Converter={x:Static controls:DoubleToLeftMarginConverter.Instance}}"`
- **잠재적 문제**: TemplateBinding과 Converter를 함께 사용할 때, 일부 WPF 버전에서 제대로 작동하지 않을 수 있음
- **대안**: 필요시 RelativeSource 바인딩으로 변경

### 2. 애니메이션 RepeatBehavior="Forever"
- **위치**: `FreshFrog39.xaml` 라인 61
- **잠재적 문제**: 컨트롤이 Unloaded될 때 애니메이션이 제대로 중지되지 않으면 메모리 누수 가능성
- **대안**: 필요시 Loaded/Unloaded 이벤트에서 Storyboard를 명시적으로 시작/중지

### 3. ease-in-out 이징 미적용
- **내용**: CSS의 `ease-in-out` 이징 함수를 WPF의 SplineDoubleKeyFrame KeySpline으로 변환하지 않음
- **현재 상태**: 선형 애니메이션으로 구현됨
- **개선 방법**: SplineDoubleKeyFrame에 `KeySpline="0.42,0 0.58,1"` 추가
