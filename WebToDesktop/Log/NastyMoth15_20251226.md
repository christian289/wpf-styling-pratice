## 에러 수정 내역

### 에러 1: StackPanel.Spacing 속성 미지원

**에러 메시지:**
```
MC3072: XML 네임스페이스 'http://schemas.microsoft.com/winfx/2006/xaml/presentation'에 'Spacing' 속성이 없습니다. 줄 15 위치 21.
```

**원인:**
WPF의 StackPanel은 Spacing 속성을 지원하지 않습니다. 이 속성은 AvaloniaUI 등 다른 XAML 플랫폼에서 사용되는 속성입니다.

**수정 방법:**
각 자식 요소에 Margin 속성을 적용하여 간격을 조절합니다.

```xml
<!-- 수정 전 -->
<StackPanel Spacing="30">
    <TextBlock ... />
</StackPanel>

<!-- 수정 후 -->
<StackPanel>
    <TextBlock ... Margin="0,0,0,30" />
</StackPanel>
```

## 잠재적 런타임 에러

### 1. 스파클 애니메이션 성능
- **설명:** 8개의 Ellipse 요소가 동시에 무한 반복 애니메이션을 실행합니다.
- **영향:** 저사양 시스템에서 성능 저하 가능성이 있습니다.
- **확인 필요:** 실제 실행 시 CPU/GPU 사용률 확인 필요

### 2. RadialGradientBrush 렌더링
- **설명:** 여러 레이어에 RadialGradientBrush가 적용되어 있습니다.
- **영향:** 소프트웨어 렌더링 환경에서 성능 저하 가능
- **확인 필요:** 다양한 하드웨어 환경에서 테스트 필요

### 3. Canvas ClipToBounds 동작
- **설명:** 체크 상태에서 KnobContainer의 ClipToBounds가 False로 변경됩니다.
- **영향:** 스파클 요소가 부모 영역을 벗어나 표시될 수 있음
- **확인 필요:** 의도된 동작인지 시각적 확인 필요

## CSS → WPF 변환 노트

| CSS 속성 | WPF 구현 |
|---------|---------|
| `@keyframes sparkle` | `Storyboard` + `DoubleAnimation` |
| `radial-gradient` | `RadialGradientBrush` |
| `border-radius: 9999px` | `CornerRadius="25"` (절반 크기) |
| `transform: translateX(...) rotate(...)` | `TransformGroup` (TranslateTransform + RotateTransform) |
| `transition: all 0.3s` | `Duration="0:0:0.3"` + `EasingFunction` |
| CSS Variables (`--primary`) | ResourceDictionary 리소스 |
| `overflow: clip` | `ClipToBounds="True"` |
| `::before`, `::after` | 별도의 Border 요소로 구현 |
