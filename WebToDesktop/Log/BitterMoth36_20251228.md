## 컴파일 에러 및 수정 내역

### 에러 1: StackPanel.Spacing 속성 미지원

**에러 내용:**
```
MainWindow.xaml(11,77): error MC3072: XML 네임스페이스 'http://schemas.microsoft.com/winfx/2006/xaml/presentation'에 'Spacing' 속성이 없습니다.
```

**원인:**
- WPF의 StackPanel은 `Spacing` 속성을 지원하지 않음
- `Spacing`은 AvaloniaUI의 StackPanel 속성임

**수정 방법:**
- StackPanel에서 `Spacing="20"` 제거
- 각 자식 요소에 `Margin="0,0,0,20"` 속성 추가하여 동일한 간격 효과 적용

**수정 전:**
```xml
<StackPanel HorizontalAlignment="Center" VerticalAlignment="Center" Spacing="20">
    <TextBlock Text="BitterMoth36 Loader" .../>
    <controls:BitterMoth36/>
    <TextBlock Text="Two circles spinning with scale animation" .../>
</StackPanel>
```

**수정 후:**
```xml
<StackPanel HorizontalAlignment="Center" VerticalAlignment="Center">
    <TextBlock Text="BitterMoth36 Loader" ... Margin="0,0,0,20"/>
    <controls:BitterMoth36 Margin="0,0,0,20"/>
    <TextBlock Text="Two circles spinning with scale animation" .../>
</StackPanel>
```

## 잠재적 런타임 에러

### 1. 애니메이션 타이밍 차이 (가능성: 낮음)
- CSS의 `ease-in-out`과 WPF의 `SineEase EasingMode="EaseInOut"`은 약간 다른 곡선을 가질 수 있음
- 시각적으로 거의 동일하나 정확한 1:1 매칭은 아님

### 2. 하단 원 애니메이션 딜레이 동기화 (가능성: 낮음)
- CSS: `animation-delay: 0.5s`로 하단 원의 scale 애니메이션이 0.5초 지연 시작
- WPF: `BeginTime="0:0:0.5"`로 구현
- 첫 사이클에서는 동일하게 동작하나, RepeatBehavior="Forever"로 반복 시 하단 원 애니메이션이 0.5초 늦게 시작됨
- 이는 CSS 원본과 동일한 동작임
