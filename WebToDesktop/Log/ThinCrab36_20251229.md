## 컴파일 에러 및 수정 내용

### Error 1: MC3024 - Rectangle.Fill 속성 중복 설정

**에러 메시지:**
```
ThinCrab36.xaml(46,46): error MC3024: 'System.Windows.Shapes.Rectangle.Fill' 속성이 이미 설정되었으며 이 속성은 한 번만 설정할 수 있습니다.
```

**원인:**
Secondary 버튼 스타일의 ShineRectangle에서 Fill 속성이 두 번 정의됨:
1. 속성 어트리뷰트: `Fill="{x:Null}"`
2. 요소 구문: `<Rectangle.Fill>...</Rectangle.Fill>`

**수정 방법:**
```xml
<!-- 수정 전 -->
<Rectangle x:Name="ShineRectangle"
           Fill="{x:Null}"
           HorizontalAlignment="Left"
           ...>
    <Rectangle.Fill>
        <LinearGradientBrush ... />
    </Rectangle.Fill>
</Rectangle>

<!-- 수정 후 -->
<Rectangle x:Name="ShineRectangle"
           HorizontalAlignment="Left"
           ...>
    <Rectangle.Fill>
        <LinearGradientBrush ... />
    </Rectangle.Fill>
</Rectangle>
```

## 잠재적 런타임 에러 (직접 확인 필요)

### 1. Typography.Capitals 속성 호환성

**위치:** `ThinCrab36.xaml` - Title TextBlock 스타일

**설명:**
```xml
<TextBlock.Style>
    <Style TargetType="TextBlock">
        <Setter Property="Typography.Capitals" Value="AllSmallCaps" />
    </Style>
</TextBlock.Style>
```

**잠재적 문제:**
- `Typography.Capitals`는 OpenType 기능을 지원하는 폰트에서만 작동
- Arial 폰트에서는 이 기능이 지원되지 않을 수 있음
- 런타임에 대문자 변환이 적용되지 않을 수 있음

**권장 대안:**
CSS의 `text-transform: uppercase`와 동일한 효과를 원하면 Converter나 Binding StringFormat 사용 고려

### 2. Shine 애니메이션 Width 바인딩

**위치:** 버튼 스타일의 ShineRectangle

**설명:**
```xml
Width="{Binding ActualWidth, RelativeSource={RelativeSource AncestorType=Border}}"
```

**잠재적 문제:**
- `ActualWidth`는 레이아웃이 완료된 후에만 유효한 값을 가짐
- 초기 로드 시 Width가 0이 되어 애니메이션이 보이지 않을 수 있음

**확인 사항:**
- 버튼 호버 시 shine 효과가 제대로 표시되는지 확인 필요

### 3. IconData Path 데이터 바인딩

**위치:** `ThinCrab36.cs` - IconData DependencyProperty

**설명:**
```csharp
public static readonly DependencyProperty IconDataProperty =
    DependencyProperty.Register(
        nameof(IconData),
        typeof(string),
        typeof(ThinCrab36),
        new PropertyMetadata("M12 2C6.48 2 ..."));
```

**잠재적 문제:**
- IconData가 `string` 타입이고 XAML에서 `Path.Data`에 바인딩됨
- 잘못된 Path 문자열이 전달되면 런타임 예외 발생 가능

**확인 사항:**
- 사용자 정의 IconData가 유효한 SVG Path 형식인지 검증 필요

|---------|---------|
