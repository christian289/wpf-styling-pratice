## Runtime Error 가능성 (직접 확인 필요)

### 1. Placeholder 표시 로직
- **위치**: `BigSloth59.xaml` - ControlTemplate.Triggers
- **내용**: Text 속성이 빈 문자열일 때와 null일 때 Placeholder를 표시하도록 여러 Trigger를 사용
- **잠재적 문제**: 텍스트 입력 후 삭제 시 Placeholder가 다시 표시되지 않을 수 있음
- **확인 방법**: 런타임에서 텍스트 입력/삭제 테스트 필요

### 2. IsFocused vs IsKeyboardFocused
- **위치**: `BigSloth59.xaml` - Focus Trigger
- **내용**: `IsFocused` 속성을 사용하여 포커스 상태 감지
- **잠재적 문제**: 일부 상황에서 `IsKeyboardFocused`가 더 정확할 수 있음
- **확인 방법**: 마우스 클릭 및 Tab 키로 포커스 이동 테스트
