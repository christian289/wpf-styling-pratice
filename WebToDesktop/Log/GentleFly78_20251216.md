## 컴파일 에러 수정

### 에러 1: StackPanel.Spacing 속성 미지원

**에러 내용:**

```
MainWindow.xaml(12,77): error MC3072: XML 네임스페이스 'http://schemas.microsoft.com/winfx/2006/xaml/presentation'에 'Spacing' 속성이 없습니다.
```

**원인:**
- WPF의 StackPanel은 Spacing 속성을 지원하지 않음
- AvaloniaUI의 StackPanel과 달리 WPF는 Spacing이 없음

**수정 방법:**
- `Spacing="20"` 속성 제거
- 자식 요소의 Margin 속성으로 간격 조정

**수정 전:**
```xml
<StackPanel VerticalAlignment="Center" HorizontalAlignment="Center" Spacing="20">
```

**수정 후:**
```xml
<StackPanel VerticalAlignment="Center" HorizontalAlignment="Center">
```

## 잠재적 런타임 오류 (직접 확인 필요)

### 1. 무한 슬라이딩 애니메이션 동기화

- **문제**: 두 패널(PART_ItemsPanel1, PART_ItemsPanel2)의 슬라이딩 애니메이션이 하드코딩된 400px 값을 사용
- **영향**: 아이템 개수나 크기에 따라 슬라이딩이 끊기거나 겹칠 수 있음
- **해결 방안**: Code-behind에서 ActualWidth를 계산하여 동적으로 애니메이션 값 설정

### 2. ItemsSource 바인딩 시 두 번째 패널 동기화

- **문제**: PART_ItemsPanel2의 ItemsControl이 TemplatedParent의 ItemsSource를 바인딩하지만, ItemsPresenter와 별도로 동작
- **영향**: 데이터 바인딩 시나리오에서 두 패널의 아이템이 일치하지 않을 수 있음
- **해결 방안**: OnApplyTemplate에서 두 패널을 동기화하는 로직 추가

### 3. Canvas 클리핑

- **문제**: Canvas.ClipToBounds가 둥근 모서리(CornerRadius)를 고려하지 않음
- **영향**: 슬라이딩 중 이모지가 둥근 모서리 바깥으로 보일 수 있음
- **해결 방안**: Border.Clip에 RectangleGeometry 적용 (RadiusX/RadiusY 설정)
