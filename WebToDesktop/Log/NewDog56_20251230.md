## 컴파일 에러

### 에러 1: StackPanel.Spacing 속성 미지원

**에러 메시지:**
```
error MC3072: XML 네임스페이스 'http://schemas.microsoft.com/winfx/2006/xaml/presentation'에 'Spacing' 속성이 없습니다.
```

**원인:**
WPF의 StackPanel은 AvaloniaUI와 달리 `Spacing` 속성을 지원하지 않습니다.

**수정 방법:**
각 자식 요소에 `Margin` 속성을 직접 적용하여 간격을 구현합니다.

```xml
<!-- 수정 전 -->
<StackPanel Spacing="20">
    <controls:NewDog56 Content="Primary button" />
</StackPanel>

<!-- 수정 후 -->
<StackPanel>
    <controls:NewDog56 Content="Primary button" Margin="0,0,0,20" />
</StackPanel>
```

## 잠재적 런타임 에러

### 1. ColorAnimation Foreground 바인딩

**위치:** `NewDog56.xaml` - Trigger.EnterActions/ExitActions

**내용:**
`Control.Foreground`의 `SolidColorBrush.Color`를 직접 애니메이션하고 있습니다. 만약 Foreground가 Frozen된 브러시이거나 SolidColorBrush가 아닌 경우 런타임 에러가 발생할 수 있습니다.

**현재 구현:**
```xml
<ColorAnimation Storyboard.TargetProperty="(Control.Foreground).(SolidColorBrush.Color)"
                To="{StaticResource NewDog56.Foreground.Hover}"
                Duration="0:0:0.2" />
```

**해결 방안 (필요시):**
템플릿 내에서 별도의 TextBlock을 사용하고 해당 Foreground를 애니메이션하거나, 컨트롤의 Foreground를 non-frozen 브러시로 설정합니다.

### 2. LinearGradientBrush RotateTransform

**위치:** `NewDog56Resources.xaml` - NewDog56.Gradient.Brush

**내용:**
CSS의 `linear-gradient(69deg, ...)` 를 WPF로 변환할 때 `RotateTransform`을 사용했습니다. CSS와 WPF의 각도 시스템이 다르므로 시각적 차이가 있을 수 있습니다.

- CSS: 0deg = 아래→위, 시계 방향
- WPF: 0deg = 왼쪽→오른쪽, 반시계 방향

**현재 구현:**
```xml
<LinearGradientBrush.RelativeTransform>
    <RotateTransform Angle="-21" CenterX="0.5" CenterY="0.5" />
</LinearGradientBrush.RelativeTransform>
```

직접 실행하여 그라데이션 방향이 원본과 일치하는지 확인이 필요합니다.
