## Runtime Error 가능성 (직접 확인 필요)

### 1. 애니메이션 BeginTime 딜레이
- **위치**: `JollyHound16.xaml` - 슬라이스 애니메이션
- **내용**: CSS `animation-delay`를 WPF `BeginTime`으로 변환
- **가능성**: 낮음 - 표준 WPF 애니메이션 패턴 사용
- **확인 방법**: 앱 실행 후 슬라이스들이 순차적으로 움직이는지 확인

### 2. TranslateTransform 퍼센트 → 픽셀 변환
- **위치**: `JollyHound16.xaml` - 각 슬라이스의 TranslateTransform
- **내용**: CSS의 `translate(5%, 5%)`를 고정 픽셀 값(8.4, 7.9)으로 변환
- **가능성**: 낮음 - SVG viewBox 기준으로 계산 (168x158)
- **확인 방법**: 슬라이스 이동 거리가 원본과 유사한지 확인

### 3. Path Data 정밀도
- **위치**: `JollyHound16.xaml` - 모든 Path 요소
- **내용**: SVG path data를 WPF Path.Data로 직접 사용
- **가능성**: 낮음 - WPF는 SVG path 문법과 호환됨
- **확인 방법**: 피자 모양이 올바르게 렌더링되는지 확인

## CSS → WPF 주요 변환 사항

| CSS 속성 | WPF 변환 |
|---------|---------|
| `animation: rotate 45s linear infinite` | `DoubleAnimation Duration="0:0:45" RepeatBehavior="Forever"` |
| `animation: slice1 4s ease-in-out infinite` | `DoubleAnimationUsingKeyFrames` + `SineEase EasingMode="EaseInOut"` |
| `animation-delay: 1s` | `BeginTime="0:0:1"` |
| `transform: translate(5%, 5%)` | `TranslateTransform X="8.4" Y="7.9"` (픽셀 계산) |
| `transform-origin: center center` | `RenderTransformOrigin="0.5,0.5"` |
| `scale: 1.6` | `Viewbox Stretch="Uniform"` + Width/Height 조정 |
| SVG `<circle>` | `Ellipse` |
| SVG `<path>` | `Path Data="..."` |
