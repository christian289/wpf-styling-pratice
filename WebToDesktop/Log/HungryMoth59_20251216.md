## 잠재적 런타임 에러 가능성

### 1. 애니메이션 타이밍 차이
- **원인**: CSS `cubic-bezier(0.3, 0.41, 0.18, 1.01)` 이징 함수를 WPF `LinearDoubleKeyFrame`으로 변환
- **영향**: 원본과 약간 다른 애니메이션 느낌
- **해결 방안**: WPF의 `SplineDoubleKeyFrame`을 사용하여 베지어 커브 적용 가능

### 2. vmax 단위 변환
- **원인**: CSS `vmax` (viewport 최대값 기반 단위)를 고정 픽셀로 변환 (1vmax ≈ 10px)
- **영향**: 원본은 뷰포트 크기에 따라 스케일링되지만, WPF 버전은 고정 크기
- **해결 방안**: Viewbox로 감싸거나 ScaleTransform 바인딩 사용

### 3. overflow: hidden + border-radius 조합
- **원인**: CSS의 paw 요소는 `overflow: hidden`과 `border-radius`로 원형 클리핑
- **영향**: WPF에서는 Canvas 내 Ellipse로 직접 구현하여 동일한 효과 달성
- **해결 방안**: 현재 구현 방식으로 충분히 유사한 시각적 효과 제공

### 4. z-index 레이어링
- **원인**: CSS z-index 값을 Panel.ZIndex와 XAML 선언 순서로 변환
- **영향**: 복잡한 레이어링에서 미세한 차이 발생 가능
- **해결 방안**: 런타임에서 확인 후 Panel.ZIndex 값 조정

### 5. 그림자 효과
- **원인**: CSS `::before` pseudo-element로 구현된 그림자를 별도 Ellipse로 변환
- **영향**: 그림자 애니메이션이 원본과 동일하게 작동
- **해결 방안**: 현재 구현으로 충분
