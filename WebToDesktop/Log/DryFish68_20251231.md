## 잠재적 런타임 오류 가능성

### 1. ColorAnimation 대상 경로 문제
- **위치**: `DryFish68.xaml` - ContentPresenter의 Foreground 애니메이션
- **코드**: `Storyboard.TargetProperty="(TextElement.Foreground).(SolidColorBrush.Color)"`
- **가능성**: ContentPresenter에 TextElement.Foreground가 직접 설정되지 않은 경우 런타임 오류 발생 가능
- **확인 필요**: 실제 실행 시 hover 애니메이션이 정상 동작하는지 확인

### 2. TranslateTransform Y 값 고정
- **위치**: `DryFish68.xaml` - BackgroundTranslate Y 애니메이션
- **코드**: `To="100"` (고정 값)
- **가능성**: 버튼 높이가 100px보다 크면 배경이 완전히 사라지지 않을 수 있음
- **개선 방안**: ActualHeight에 바인딩하는 Converter 사용 고려

### 3. ClipToBounds 의존성
- **위치**: `DryFish68.xaml` - OuterBorder
- **설명**: `ClipToBounds="True"`로 overflow hidden 효과 구현
- **가능성**: Border의 CornerRadius와 ClipToBounds 조합 시 일부 WPF 버전에서 둥근 모서리 클리핑이 완벽하지 않을 수 있음
- **개선 방안**: 필요시 Border.Clip + RectangleGeometry 패턴 적용

## CSS → WPF 변환 매핑

| CSS 속성 | WPF 구현 |
|---------|---------|
| `border-radius: 50px` | `CornerRadius="50"` |
| `border: 2px solid #ed553b` | `BorderThickness="2"`, `BorderBrush` |
| `padding: 14px 40px 13px` | `Padding="40,14,40,13"` |
| `overflow: hidden` | `ClipToBounds="True"` |
| `::before` pseudo-element | Rectangle 요소로 구현 |
| `transition: all .3s ease` | DoubleAnimation + CubicEase |
| `height: 111% -> 11%` | TranslateTransform Y 이동 |
| `z-index` | XAML 선언 순서 (먼저 선언 = 뒤에 배치) |
