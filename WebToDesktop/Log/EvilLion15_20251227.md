## 컴파일 에러

### 에러 1: MC3044 - MarkupExtension 파싱 오류

**에러 내용:**
```
error MC3044: MarkupExtension 식의 닫는 '}' 뒤에 텍스트 ',0,0,0'을(를) 사용할 수 없습니다.
```

**발생 위치:**
- `EvilLion15.Wpf.UI\Themes\EvilLion15.xaml` Line 30, 49

**원인:**
WPF XAML에서 `Margin` 속성에 `StaticResource`와 리터럴 값을 혼합하여 사용할 수 없음.

잘못된 예:
```xml
Margin="{StaticResource EvilLion15.Indicator.LeftPosition},0,0,0"
```

**수정 방법:**
1. `EvilLion15Resources.xaml`에서 `sys:Double` 타입의 단일 값 대신 `Thickness` 타입의 전체 Margin 값으로 변경

변경 전:
```xml
<sys:Double x:Key="EvilLion15.Indicator.LeftPosition">-23</sys:Double>
<sys:Double x:Key="EvilLion15.Indicator.RightPosition">-23</sys:Double>
```

변경 후:
```xml
<Thickness x:Key="EvilLion15.LeftIndicator.Margin">-23,0,0,0</Thickness>
<Thickness x:Key="EvilLion15.RightIndicator.Margin">0,0,-23,0</Thickness>
```

2. `EvilLion15.xaml`에서 수정된 리소스 키 사용

변경 전:
```xml
Margin="{StaticResource EvilLion15.Indicator.LeftPosition},0,0,0"
Margin="0,0,{StaticResource EvilLion15.Indicator.RightPosition},0"
```

변경 후:
```xml
Margin="{StaticResource EvilLion15.LeftIndicator.Margin}"
Margin="{StaticResource EvilLion15.RightIndicator.Margin}"
```

## 잠재적 런타임 오류

### 1. 인디케이터 클리핑 문제
- **가능성:** 낮음
- **설명:** 인디케이터가 스위치 트랙 외부에 위치하여 음수 Margin을 사용함. 부모 컨테이너에 `ClipToBounds="True"`가 설정되어 있으면 인디케이터가 보이지 않을 수 있음.
- **확인 필요:** Gallery 앱 실행 시 인디케이터 점이 양쪽에 표시되는지 확인

### 2. 토글 애니메이션 상태 동기화
- **가능성:** 낮음
- **설명:** `IsChecked="True"`로 초기화된 컨트롤의 경우 초기 Thumb 위치가 체크되지 않은 상태(왼쪽)로 표시된 후 체크된 위치로 이동할 수 있음.
- **확인 필요:** 초기 `IsChecked="True"` 컨트롤이 오른쪽 위치에서 바로 렌더링되는지 확인

### 3. RadialGradientBrush 렌더링
- **가능성:** 매우 낮음
- **설명:** CSS의 `radial-gradient(45%, circle, ...)` 형식을 WPF `RadialGradientBrush`로 변환. `GradientOrigin`과 `Center`가 동일한 값(0.45,0.5)으로 설정됨.
- **확인 필요:** 인디케이터의 그라디언트가 원본 CSS와 유사하게 표시되는지 확인
