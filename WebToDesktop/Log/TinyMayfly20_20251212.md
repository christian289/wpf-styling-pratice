## 변환 시 주요 결정 사항

### 1. 픽셀 아트 캐릭터 변환

**원본 CSS**: `box-shadow`를 사용한 픽셀 단위 아트 (수백 개의 픽셀 좌표)

**WPF 구현**: 기본 도형(Ellipse, Rectangle, Polygon)을 사용한 단순화된 캐릭터 실루엣

**이유**: CSS box-shadow로 구현된 픽셀 아트를 WPF로 1:1 변환하면:
- 수백 개의 Rectangle 요소 필요
- 렌더링 성능 저하
- 유지보수 어려움

단순화된 벡터 그래픽으로 캐릭터의 실루엣을 표현하여 원본의 느낌을 유지하면서 성능을 확보함.

### 2. 애니메이션 변환

| CSS | WPF |
|-----|-----|
| `@keyframes head` (top 애니메이션) | `DoubleAnimationUsingKeyFrames` + `TranslateTransform.Y` |
| `transition: 0.5s` | `DoubleAnimation Duration="0:0:0.3"` |
| `:hover` 상태 | `EventTrigger` (MouseEnter/MouseLeave) |

### 3. 레이아웃 변환

- `.notification` (250x60) → `Border` with Background
- `.parchment` (100x100) → `Border` with dashed border
- `.text` (flex column) → `StackPanel`

## 잠재적 Runtime Error 가능성

### 1. VisualState 애니메이션 충돌 (낮음)
- **상황**: `VisualStateManager`의 `Normal`/`MouseOver` 상태와 `EventTrigger`가 동시에 작동
- **영향**: 애니메이션이 부드럽지 않을 수 있음
- **해결**: 현재 `EventTrigger`만 사용하도록 구현되어 문제 없음

### 2. 폰트 미설치 (낮음)
- **상황**: `Consolas` 폰트가 시스템에 없는 경우
- **영향**: 기본 폰트로 대체 표시
- **해결**: Fallback으로 `Courier New, monospace` 지정됨

### 3. 캐릭터 영역 클리핑 (낮음)
- **상황**: Canvas의 ScaleTransform으로 인해 일부 요소가 부모 영역을 벗어날 수 있음
- **영향**: 시각적 잘림 현상
- **해결**: 충분한 마진과 컨트롤 크기 확보
