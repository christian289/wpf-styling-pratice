# OrangeNewt83 변환 로그

## 변환 정보

- **변환 일시**: 2025-12-29
- **원본**: HTML/CSS (uiverse.io)
- **대상**: WPF CustomControl

## 컴파일 에러 및 수정 사항

### 에러 1: IsPressed 속성 없음

**에러 메시지**:
```
MC4109: 'OrangeNewt83.Wpf.UI.Controls.OrangeNewt83' 형식에서 템플릿 속성 'IsPressed'을(를) 찾을 수 없습니다.
```

**원인**:
- `ContentControl`을 상속받은 컨트롤에서는 `IsPressed` 속성을 사용할 수 없음
- `IsPressed`는 `ButtonBase` 클래스에서만 제공되는 속성

**수정 방법**:
- CSS `:active` 상태에 대응하는 `IsPressed` Trigger를 제거함
- 원본 CSS의 `:active` 효과(그림자 제거)는 WPF에서 구현되지 않음

**수정 코드**:
```xml
<!-- 제거된 Trigger -->
<Trigger Property="IsPressed" Value="True">
    <Setter TargetName="BackCard" Property="Effect" Value="{x:Null}"/>
    <Setter TargetName="FrontCard" Property="Effect" Value="{x:Null}"/>
</Trigger>
```

## 잠재적 런타임 에러

### 1. DropShadowEffect 성능 이슈 (가능성: 낮음)

**설명**:
- `DropShadowEffect`는 소프트웨어 렌더링을 사용하므로 많은 카드가 동시에 표시될 경우 성능 저하 가능

**해결 방안**:
- 필요시 `Effect`를 비활성화하거나 `BitmapCache`를 사용

### 2. 회전 애니메이션 클리핑 (가능성: 낮음)

**설명**:
- 회전된 카드가 부모 컨테이너에 의해 클리핑될 수 있음

**현재 대응**:
- `ClipToBounds="False"` 설정으로 방지

### 3. CSS `:active` 상태 미구현 (가능성: 해당 없음)

**설명**:
- 원본 CSS의 `:active` 상태(클릭 시 그림자 제거)가 구현되지 않음
- `ContentControl`에서 `IsPressed` 사용 불가

**대안**:
- 필요시 `PreviewMouseDown`/`PreviewMouseUp` 이벤트 핸들러 사용
- 또는 `ButtonBase` 상속으로 변경
