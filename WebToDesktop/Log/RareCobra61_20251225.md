## 잠재적 런타임 에러 (확인 필요)

### 1. PasswordBox 바인딩 제한

**문제**: WPF의 `PasswordBox.Password` 속성은 보안상 `DependencyProperty`가 아니므로 직접 바인딩이 불가능합니다.

**현재 구현**: `PasswordBoxHelper.HasPassword` 첨부 속성으로 플레이스홀더 표시만 처리했습니다. 실제 비밀번호 값을 가져오려면 코드 비하인드에서 처리해야 합니다.

**해결 방안**:
```csharp
// MainWindow.xaml.cs에서 PART_PasswordBox에 접근하여 Password 값 사용
var passwordBox = (PasswordBox)loginControl.Template.FindName("PART_PasswordBox", loginControl);
string password = passwordBox.Password;
```

### 2. Google 아이콘 크기 조정

**문제**: Google 아이콘은 48x48 viewBox를 사용하며, 18px 크기로 표시하기 위해 `ScaleTransform`을 적용했습니다.

**위험도**: 낮음 - 렌더링 시 스케일 계산이 정확하지 않을 수 있음

### 3. 폰트 가용성

**문제**: "Lucida Sans" 폰트 패밀리가 시스템에 없을 경우 대체 폰트가 사용됩니다.

**위험도**: 낮음 - WPF가 자동으로 대체 폰트를 선택함

## CSS → WPF 특이 변환 사항

| CSS | WPF 변환 | 비고 |
|-----|----------|------|
| `gap: 18px` | 각 요소에 `Margin="0,0,0,18"` | WPF는 gap을 지원하지 않음 |
| `::placeholder` 스타일링 | `Tag` 속성 + Trigger | TextBox placeholder 구현 |
| SVG 멀티컬러 | Canvas + 다중 Path | Google 아이콘의 4가지 색상 유지 |
| `box-shadow` 복합 | 단일 `DropShadowEffect` | WPF는 다중 그림자 미지원 |
| `font-weight: 800` | `FontWeight="ExtraBold"` | 숫자 → 열거형 변환 |
