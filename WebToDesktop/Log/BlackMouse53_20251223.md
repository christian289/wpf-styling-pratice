## Runtime Error 가능성

### 1. RadialGradientBrush 렌더링 차이

**잠재적 문제**: CSS의 `radial-gradient`와 WPF의 `RadialGradientBrush`는 렌더링 방식이 다릅니다.

- CSS: `radial-gradient(circle at -100% 90%, ...)` - 원점이 요소 밖에 있을 수 있음
- WPF: `GradientOrigin` 값이 0~1 범위를 벗어날 때 예상과 다르게 렌더링될 수 있음

**확인 필요**:
- 날개(Wing) 요소들이 CSS 원본과 동일하게 보이는지 시각적 확인 필요
- 특히 `GradientOrigin="-1,0.9"` 같은 범위 밖 값들의 렌더링 결과

### 2. 레이아웃 클리핑

**잠재적 문제**: Canvas에 `ClipToBounds="False"`를 설정했으나, 부모 요소에서 클리핑될 수 있음

- Beak 요소들이 `Canvas.Top="-36"`, `Canvas.Top="-33"`으로 Canvas 영역 밖에 위치
- 부모 Grid나 Window에서 클리핑되면 부리가 잘려 보일 수 있음

**확인 필요**:
- 부리(Beak) 부분이 완전히 보이는지 확인
- 필요시 Window 또는 Container의 Margin 조정

### 3. 색상 커스터마이징

**주의사항**: `BirdColor` DependencyProperty가 정의되어 있지만, 현재 스타일에서는 리소스의 고정 색상만 사용 중

- `TemplateBinding`을 통한 동적 색상 변경이 필요하면 추가 작업 필요
