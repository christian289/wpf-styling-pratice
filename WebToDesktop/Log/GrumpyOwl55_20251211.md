## 컴파일 에러 및 수정 내용

### 에러 1: XML 주석에서 '--' 사용 불가
**에러 메시지:**
```
MC3000: 'An XML comment cannot contain '--', and '-' cannot be the last character.'
```

**원인:**
- XAML 주석(`<!-- -->`) 내에서 CSS 변수명(`--sz`, `--c0`, `--c1` 등)을 그대로 사용
- XML 표준에서 주석 내 `--` 시퀀스는 허용되지 않음

**수정 방법:**
- CSS 변수명에서 앞의 `--` 접두사를 제거
- 예: `--sz` → `sz`, `--c0` → `c0`

**수정 파일:**
1. `GrumpyOwl55Resources.xaml`
2. `GrumpyOwl55.xaml`

## 잠재적 런타임 오류 (확인 필요)

### 1. conic-gradient 근사
- CSS의 `conic-gradient`는 WPF에서 직접 지원되지 않음
- PathGeometry 직사각형으로 근사 처리
- 원본과 시각적 차이가 있을 수 있음

### 2. 복잡한 패턴 레이어링
- CSS는 20개 이상의 gradient 레이어를 중첩
- WPF에서 DrawingGroup으로 레이어를 구현했으나, 일부 레이어가 단순화됨
- 특히 `linear-gradient(-45deg, ...)` 오버레이가 정확히 일치하지 않을 수 있음

### 3. 타일링 경계
- 타일 경계에서 도트 패턴이 잘릴 수 있음
- EllipseGeometry가 타일 경계(0, 240)에 위치하여 반만 보일 수 있음

## CSS → WPF 변환 매핑

| CSS 속성 | WPF 구현 |
|---------|----------|
| `repeating-linear-gradient(90deg, ...)` | 중첩 DrawingBrush TileMode="Tile" |
| `linear-gradient(-45deg, ...)` | LinearGradientBrush StartPoint="1,0" EndPoint="0,1" |
| `conic-gradient(from -90deg at X% Y%, ...)` | PathGeometry (직사각형 근사) |
| `radial-gradient(circle at X% Y%, ...)` | EllipseGeometry |
| CSS 변수 (`--sz`, `--c0`, `--c1`) | StaticResource (Colors, Brushes) |
