## 컴파일 에러 및 수정 내용

### 에러 1: TextBlock.CharacterSpacing 속성 미지원
- **에러 메시지**: `MC3072: XML 네임스페이스 'http://schemas.microsoft.com/winfx/2006/xaml/presentation'에 'TextBlock.CharacterSpacing' 속성이 없습니다.`
- **원인**: WPF에서는 `TextBlock.CharacterSpacing` 속성이 지원되지 않음. 이 속성은 UWP/WinUI에서만 사용 가능
- **수정 방법**: `ContentPresenter`에서 해당 속성 제거
- **CSS 원본**: `letter-spacing: 0.1em;`
- **영향**: 글자 간격 효과가 적용되지 않음. 시각적으로 미미한 차이

### 에러 2: StackPanel.Spacing 속성 미지원 (Gallery 프로젝트)
- **에러 메시지**: `StackPanel`에 `Spacing` 속성 없음
- **원인**: WPF에서는 `StackPanel.Spacing` 속성이 지원되지 않음. 이 속성은 UWP/WinUI에서만 사용 가능
- **수정 방법**: 각 요소에 `Margin` 속성으로 간격 지정
- **영향**: 없음. 동일한 결과

## 잠재적 Runtime Error 가능성

### 1. ColorAnimation 타겟 경로 문제
- **위치**: `WiseMule92.xaml` - `Storyboard.TargetProperty="(TextBlock.Foreground).(SolidColorBrush.Color)"`
- **가능성**: ContentPresenter가 TextBlock이 아닌 다른 요소를 포함할 경우 애니메이션이 동작하지 않을 수 있음
- **확인 필요**: 런타임에서 텍스트 외 Content 사용 시 테스트 필요

### 2. CSS box-shadow 완전 변환 불가
- **원본 CSS**: 복합 inset/outset box-shadow (8개의 그림자 레이어)
- **WPF 구현**: 단일 `DropShadowEffect`로 단순화
- **차이점**: 원본 CSS의 복잡한 그림자 효과가 단순화됨

## CSS → WPF 변환 매핑

| CSS 속성 | WPF 구현 |
|----------|----------|
| `background-color: #f44336` | `SolidColorBrush` |
| `color: #fff` | `Foreground` / `TextBlock.Foreground` |
| `padding: 10px 20px` | `Padding="20,10,20,10"` |
| `font-size: 20px` | `FontSize="20"` |
| `font-weight: bold` | `FontWeight="Bold"` |
| `text-transform: uppercase` | 직접 적용 불가 (Content에 대문자로 입력) |
| `letter-spacing: 0.1em` | 미지원 (제거됨) |
| `cursor: pointer` | `Cursor="Hand"` |
| `transition: all 0.3s ease-in-out` | `Storyboard` + `DoubleAnimation` / `ColorAnimation` |
| `:hover` 배경색 변경 | `Trigger` + `ColorAnimation` |
| `:hover span transform: rotate(-10deg) scale(1.1)` | `RotateTransform` + `ScaleTransform` 애니메이션 |
| `box-shadow` (복합) | `DropShadowEffect` (단순화) |
