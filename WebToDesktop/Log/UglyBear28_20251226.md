## 컴파일 에러

### 에러 1: XML 주석 내 '--' 사용 오류

**에러 메시지:**
```
MC3000: 'An XML comment cannot contain '--', and '-' cannot be the last character. Line 17, position 9.' XML이 잘못되었습니다.
```

**원인:**
- XAML 주석 내에 CSS 변수명 `--mask`를 그대로 사용
- XML 주석에서 `--`는 허용되지 않음

**수정 방법:**
- CSS 변수 `--mask`를 `[mask]`로 변경
- `-webkit-mask`를 `webkit-mask`로 변경
- 불렛 포인트 `-`를 제거하고 일반 텍스트로 변경

**수정 전:**
```xml
<!--
    --mask: linear-gradient(red, rgba(0, 0, 0, 0.45));
    -webkit-mask: var(--mask);
    - filter: contrast(7)는 WPF에서 직접 지원하지 않음
-->
```

**수정 후:**
```xml
<!--
    [mask]: linear-gradient(red, rgba(0, 0, 0, 0.45));
    webkit-mask: var([mask]);
    filter: contrast(7)는 WPF에서 직접 지원하지 않음
-->
```

## 런타임 오류 가능성

### 1. CSS filter: contrast(7) 미지원
- **상태:** WPF에서 직접 지원하지 않음
- **영향:** 원본 CSS와 시각적 차이 발생 가능
- **원본 효과:** 높은 대비로 점 패턴이 더 선명하게 보임
- **현재 구현:** 대비 필터 없이 점 패턴만 표시

### 2. OpacityMask 색상 해석
- **상태:** CSS mask와 WPF OpacityMask의 동작 방식 차이
- **CSS:** `linear-gradient(red, rgba(0, 0, 0, 0.45))` - 색상 기반 마스킹
- **WPF:** OpacityMask는 알파 채널만 사용하여 투명도 결정
- **영향:** 마스킹 효과가 원본과 다를 수 있음

### 3. 패턴 크기 고정값
- **상태:** CSS `1em`을 고정값 `16px`로 변환
- **영향:** 폰트 크기와 연동되지 않음
- **권장:** 필요시 DependencyProperty로 패턴 크기 조절 기능 추가
