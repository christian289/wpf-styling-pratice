## CSS → WPF 변환 분석

### 원본 CSS 특징
원본 CSS는 복잡한 패턴을 생성하기 위해 다음 그라데이션을 조합합니다:

1. `repeating-linear-gradient(45deg)` - 대각선 흰색 라인
2. `repeating-linear-gradient(-45deg)` - 반대 대각선 흰색 라인
3. `repeating-linear-gradient(0deg)` - 수직 흰색 라인
4. `repeating-linear-gradient(90deg)` - 수평 흰색 라인
5. `conic-gradient` (4개) - 각 모서리에 골드 그라데이션
6. `repeating-conic-gradient` - Teal/Red 체커보드 베이스

### WPF 구현 전략
- **conic-gradient 미지원**: WPF는 conic gradient를 직접 지원하지 않음
- **DrawingBrush 사용**: `DrawingBrush` + `GeometryDrawing` 조합으로 패턴 구현
- **PathGeometry**: 삼각형/사각형 형태로 conic gradient 효과 근사
- **TileMode**: `DrawingBrush.TileMode="Tile"`로 반복 패턴 생성

## 잠재적 런타임 오류

### 1. 성능 관련 (Performance)
- **복잡한 DrawingBrush**: 9개의 GeometryDrawing과 다수의 LineGeometry로 구성
- **대형 화면에서 성능 저하 가능성**: 타일이 많아질수록 렌더링 부하 증가
- **권장 사항**: 대형 디스플레이에서 TileSize 값을 크게 설정

### 2. 시각적 차이 (Visual Differences)
- **conic-gradient 근사**: 원본 CSS의 원뿔형 그라데이션을 삼각형으로 근사하여 시각적 차이 발생 가능
- **라인 정밀도**: 1px 라인이 DPI 스케일링에 따라 다르게 렌더링될 수 있음

### 3. TileSize 바인딩 미적용 (Not Implemented)
- **현재 상태**: TileSize DependencyProperty가 정의되어 있으나 Viewport에 동적 바인딩되지 않음
- **영향**: TileSize 변경 시 패턴 크기가 변하지 않음
- **해결 방법**: Converter 또는 MultiBinding을 사용하여 Viewport를 TileSize에 바인딩 필요
