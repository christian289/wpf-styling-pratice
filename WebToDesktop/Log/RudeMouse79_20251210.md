## 컴파일 에러

### 에러 1: StackPanel.Spacing 속성 미지원

**에러 메시지**:
```
MainWindow.xaml(11,77): error MC3072: XML 네임스페이스 'http://schemas.microsoft.com/winfx/2006/xaml/presentation'에 'Spacing' 속성이 없습니다.
```

**원인**:
- WPF의 `StackPanel`은 `Spacing` 속성을 지원하지 않음
- `Spacing`은 Avalonia UI 또는 WinUI에서 사용하는 속성

**수정 방법**:
```xml
<!-- 수정 전 -->
<StackPanel Spacing="20">

<!-- 수정 후 -->
<StackPanel>
    <controls:RudeMouse79 ... />
    <controls:RudeMouse79 Margin="0,20,0,0" ... />
</StackPanel>
```

**대안**:
- 각 자식 요소에 `Margin` 속성 적용
- 또는 `UniformGrid`, `WrapPanel` 등 다른 패널 사용

## 잠재적 런타임 에러

### 1. ColorAnimation의 Transparent 대상

**위치**: `RudeMouse79.xaml` - RudeMouse79Item 스타일의 IsSelected Trigger

**코드**:
```xml
<ColorAnimation To="Transparent" ... />
```

**잠재적 문제**:
- `Transparent`는 `Color.FromArgb(0, 255, 255, 255)`로 해석됨
- 일부 상황에서 색상 보간 시 예상치 못한 결과 발생 가능

**권장 확인사항**:
- 선택 해제 시 배경색 전환 애니메이션이 자연스럽게 동작하는지 확인
- 문제 발생 시 `#00EEEEEE` (컨테이너 배경색의 투명 버전)로 변경 권장

### 2. Selector 기반 컨트롤의 아이템 선택 동작

**위치**: `RudeMouse79.cs` - Selector 상속

**잠재적 문제**:
- `Selector` 클래스는 기본적으로 단일 선택만 지원
- `RudeMouse79Item`이 `ListBoxItem`을 상속하지만, 부모 컨트롤이 `ListBox`가 아니므로 일부 동작 차이 가능

**권장 확인사항**:
- 아이템 클릭 시 `IsSelected` 상태가 올바르게 토글되는지 확인
- 기존 선택이 해제되고 새 선택이 적용되는지 확인
- 키보드 네비게이션 (화살표 키) 동작 확인

### 3. ResourceDictionary 병합 순서

**위치**: `Generic.xaml`, `RudeMouse79.xaml`

**잠재적 문제**:
- `RudeMouse79Resources.xaml`의 리소스가 로드되기 전에 참조될 경우 런타임 에러 발생 가능

**현재 구조** (올바름):
```
Generic.xaml
  └─ RudeMouse79.xaml
       └─ RudeMouse79Resources.xaml
```

**권장 확인사항**:
- 애플리케이션 시작 시 XAML 파싱 에러 없이 로드되는지 확인
