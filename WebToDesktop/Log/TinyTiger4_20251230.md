## 잠재적 Runtime Error 가능성

### 1. DrawingBrush 렌더링 이슈
- **위치**: `TinyTiger4.xaml` - BubbleCircle의 Fill 속성
- **설명**: CSS의 다중 radial-gradient를 DrawingGroup으로 변환했으나, WPF에서 DrawingBrush의 좌표계가 다를 수 있음
- **확인 필요**: 버블 효과의 그라데이션이 원본 CSS와 동일하게 렌더링되는지 실행하여 확인 필요

### 2. 애니메이션 초기 상태 이슈
- **위치**: `TinyTiger4.xaml` - ControlTemplate.Triggers
- **설명**: 컨트롤 로드 시 IsChecked="False"인 경우 Trigger.EnterActions가 즉시 실행될 수 있음
- **가능한 증상**: 초기 로드 시 버블이 잠깐 나타났다 사라지는 현상
- **확인 필요**: 체크되지 않은 상태로 시작하는 라디오 버튼의 초기 렌더링 확인 필요

### 3. ScaleTransform 애니메이션 타이밍
- **위치**: `TinyTiger4.xaml` - IsChecked=False 트리거의 DoubleAnimationUsingKeyFrames
- **설명**: CSS의 animation이 "버블이 커졌다 사라지는" 효과인데, KeyTime이 동일한 0:0:0.5에서 scale(1.7)과 scale(0)이 동시에 발생
- **가능한 증상**: 애니메이션이 예상대로 동작하지 않을 수 있음
- **확인 필요**: Unchecked 애니메이션이 CSS 원본과 동일하게 동작하는지 확인 필요

## CSS → WPF 변환 특이사항

### radial-gradient 다중 레이어
CSS 원본에서 `background` 속성에 4개의 radial-gradient가 레이어로 쌓여있음:
1. 흰색 하이라이트 (circle at 70% 30%)
2. 마젠타 외곽 글로우 (circle at 60% 55%)
3. 시안 링 (circle at 50% 50%)
4. 노랑 링 (circle at 50% 55%)

WPF에서는 `DrawingBrush` + `DrawingGroup`을 사용하여 각 레이어를 `GeometryDrawing`으로 구현함.
그러나 CSS의 정확한 그라데이션 범위와 WPF의 좌표계 차이로 시각적 결과가 다를 수 있음.

### ::before / ::after pseudo-element
- `::before`: 배경 원형 (BackgroundCircle Ellipse)
- `::after`: 버블 효과 (BubbleCircle Ellipse with ScaleTransform)

두 요소를 Grid 내에 배치하여 겹쳐지도록 구현함.
