## 잠재적 런타임 에러 가능성

### 1. clip-path 폴리곤 좌표 고정값
- **문제**: CSS에서는 퍼센트(%) 기반 좌표를 사용하지만, WPF PathGeometry에서는 절대 픽셀 좌표로 변환함
- **현재 구현**: 190x254 크기 기준으로 고정된 좌표값 사용
- **영향**: 컨트롤 크기를 변경하면 clip-path가 비율에 맞지 않게 될 수 있음
- **해결 방법**: MultiValueConverter를 사용하여 ActualWidth/ActualHeight 기반 동적 좌표 계산 필요

### 2. backdrop-filter 미지원
- **문제**: CSS `backdrop-filter: blur(20px)`는 WPF에서 직접 지원되지 않음
- **현재 구현**: 단순 반투명 배경(`rgba(255,255,255,0.074)`)으로 대체
- **영향**: 원본과 시각적 차이가 있음 (블러 효과 없음)
- **해결 방법**: BlurEffect + OpacityMask 조합 또는 Win2D/Composition API 활용 필요

### 3. 텍스트 클리핑 영역
- **문제**: 클립 경로가 변형되는 동안 텍스트가 잘릴 수 있음
- **현재 구현**: 텍스트가 ClippedContent Border 내부에 배치되어 clip-path의 영향을 받음
- **영향**: 초기 상태에서 텍스트 일부가 보이지 않을 수 있음 (의도된 동작)

## CSS → WPF 변환 매핑

| CSS | WPF |
|-----|-----|
| `clip-path: polygon(...)` | `Border.Clip` + `PathGeometry` with `LineSegment` |
| `linear-gradient(135deg, ...)` | `LinearGradientBrush` StartPoint="0,0" EndPoint="1,1" |
| `backdrop-filter: blur(20px)` | 미지원 (반투명 배경으로 대체) |
| `transform: rotate(34deg)` | `RotateTransform` |
| `transition: 0.5s` | `PointAnimation`, `ThicknessAnimation`, `DoubleAnimation` Duration="0:0:0.5" |
| `:hover` | `Trigger Property="IsMouseOver"` |
| `opacity: 0` → `opacity: 1` | `DoubleAnimation` Opacity 0 → 1 |
| `margin-left: 25%` | `Margin="47.5,0,0,0"` (190 * 0.25 = 47.5) |
| `padding-top: 6.3em` | `Margin="0,100.8,0,0"` (16 * 6.3 = 100.8) |
