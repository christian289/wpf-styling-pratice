## 컴파일 에러 및 수정

### 에러 1: StackPanel.Spacing 속성 없음

**에러 내용:**
```
MC3072: XML 네임스페이스 'http://schemas.microsoft.com/winfx/2006/xaml/presentation'에 'Spacing' 속성이 없습니다.
```

**원인:**
- WPF의 StackPanel에는 `Spacing` 속성이 없음 (AvaloniaUI에만 존재)

**수정 방법:**
- `Spacing="30"` 속성 제거
- 필요시 각 자식 요소에 `Margin` 속성으로 간격 지정

**수정 전:**
```xml
<StackPanel HorizontalAlignment="Center"
            VerticalAlignment="Center"
            Spacing="30">
```

**수정 후:**
```xml
<StackPanel HorizontalAlignment="Center"
            VerticalAlignment="Center">
```

## 런타임 에러 가능성 (직접 확인 필요)

### 1. Inset Box-Shadow 시뮬레이션
- CSS의 `box-shadow: inset` 효과를 LinearGradientBrush로 시뮬레이션
- 원본과 시각적으로 약간 다를 수 있음

### 2. Neumorphism 이중 그림자
- CSS는 `box-shadow`에 여러 그림자를 동시에 지정 가능
- WPF는 DropShadowEffect 하나만 적용 가능하므로 두 개의 Border를 겹쳐서 구현
- 그림자가 겹치는 부분에서 시각적 차이 발생 가능

### 3. 애니메이션 Easing
- CSS `ease-in`을 WPF `QuadraticEase EaseMode="EaseIn"`으로 변환
- 미세한 타이밍 차이 가능

## CSS → WPF 변환 매핑

| CSS | WPF |
|-----|-----|
| `width: 60px; height: 30px` | `Width="60" Height="30"` |
| `background: #d3d3d3` | `Background="#D3D3D3"` |
| `border-radius: 50px` | `CornerRadius="15"` (높이의 절반) |
| `box-shadow: 9px 9px 29px #969696` | `DropShadowEffect BlurRadius="29" ShadowDepth="9" Direction="315"` |
| `box-shadow: -9px -9px 29px #ffffff` | `DropShadowEffect BlurRadius="29" ShadowDepth="9" Direction="135"` |
| `box-shadow: inset ...` | `LinearGradientBrush` (시뮬레이션) |
| `transition: all 0.3s ease-in` | `Duration="0:0:0.3"` + `QuadraticEase EaseMode="EaseIn"` |
| `transform: rotate(220deg)` | `RotateTransform Angle="220"` |
| `left: 29px` | `TranslateTransform X="30"` |
| `:checked` selector | `<Trigger Property="IsChecked" Value="True">` |
