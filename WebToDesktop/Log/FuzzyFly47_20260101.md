## 컴파일 에러 및 수정 내역

### 에러 1: Typography 속성 설정 오류

**에러 메시지:**
```
error MC3065: 'Typography' 속성은 읽기 전용이며 태그에서 설정할 수 없습니다.
```

**원인:**
- CSS `text-transform: uppercase`를 WPF로 변환 시 `<TextBlock.Typography><Typography Capitals="AllSmallCaps"/></TextBlock.Typography>` 형태로 작성
- WPF에서 Typography는 읽기 전용 속성이므로 자식 요소로 설정 불가

**수정 방법:**
- Typography를 attached property 형식으로 변경
- `<TextBlock.Typography>` 제거하고 `Typography.Capitals="AllSmallCaps"` 속성으로 직접 설정

**수정 전:**
```xml
<TextBlock Text="{TemplateBinding Message}">
    <TextBlock.Typography>
        <Typography Capitals="AllSmallCaps"/>
    </TextBlock.Typography>
</TextBlock>
```

**수정 후:**
```xml
<TextBlock Text="{TemplateBinding Message}"
           Typography.Capitals="AllSmallCaps"/>
```

## 잠재적 Runtime 오류 가능성

### 1. Typography.Capitals 글꼴 호환성
- **상황**: `Typography.Capitals="AllSmallCaps"`는 OpenType 기능을 지원하는 글꼴에서만 작동
- **증상**: 지원하지 않는 글꼴 사용 시 대문자 변환이 적용되지 않음
- **확인 필요**: 다양한 글꼴로 테스트하여 대문자 변환 확인

### 2. CSS inset box-shadow 시뮬레이션
- **상황**: CSS의 `box-shadow: inset`을 중첩 Border로 구현
- **증상**: 원본 CSS와 완벽히 동일한 그림자 효과가 아닐 수 있음
- **확인 필요**: 시각적 비교 필요

### 3. 애니메이션 타이밍
- **상황**: CSS `ease-in-out`을 WPF `SplineDoubleKeyFrame`의 `KeySpline="0.42,0,0.58,1"`로 변환
- **증상**: CSS와 미세한 타이밍 차이 발생 가능
- **확인 필요**: 애니메이션 부드러움 확인
