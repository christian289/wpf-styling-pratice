## Runtime Error 가능성 (직접 확인 필요)

### 1. SVG Path 렌더링
- **위험도**: 낮음
- **설명**: 원본 SVG의 `d="M12 19l9 2-9-18-9 18 9-2zm0 0v-8"` 경로를 WPF PathGeometry의 LineSegment로 변환함. 원본과 시각적으로 약간 다를 수 있음.
- **확인 방법**: 실행 후 아이콘 모양이 비행기처럼 보이는지 확인

### 2. Canvas 내 아이콘 회전
- **위험도**: 낮음
- **설명**: CSS `rotate-45`를 WPF `RotateTransform`으로 변환. CenterX/CenterY를 12,12로 설정하여 중심 기준 회전.
- **확인 방법**: 아이콘이 45도 회전되어 보이는지 확인

### 3. Hover 애니메이션
- **위험도**: 낮음
- **설명**: CSS `transition duration-300 hover:scale-110`을 WPF Storyboard 애니메이션으로 변환. ScaleTransform 사용.
- **확인 방법**: 마우스 호버 시 버튼이 부드럽게 확대되는지 확인

## Tailwind CSS → WPF 변환 매핑

| Tailwind CSS 클래스 | WPF 구현 |
|---|---|
| `bg-white` | `Background="White"` |
| `rounded-full` | `CornerRadius="9999"` |
| `px-4 py-2` | `Padding="16,8"` |
| `shadow-md` | `DropShadowEffect` (BlurRadius=6, ShadowDepth=4) |
| `hover:scale-110` | `ScaleTransform` + `Storyboard` 애니메이션 |
| `transition duration-300` | `Duration="0:0:0.3"` |
| `rotate-45` | `RotateTransform Angle="45"` |
| `font-semibold` | `FontWeight="SemiBold"` |
| `text-black` | `Foreground="Black"` |
| `space-x-2` | `Margin="0,0,8,0"` |
| `cursor-pointer` | `Cursor="Hand"` |
