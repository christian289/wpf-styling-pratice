## 컴파일 에러 및 수정

### 에러 1: StackPanel.Spacing 속성 미지원

**에러 내용:**
```
error MC3072: XML 네임스페이스 'http://schemas.microsoft.com/winfx/2006/xaml/presentation'에 'Spacing' 속성이 없습니다.
```

**원인:**
WPF의 StackPanel은 Spacing 속성을 지원하지 않습니다. Spacing은 AvaloniaUI 전용 속성입니다.

**수정 방법:**
각 자식 요소에 Margin 속성을 적용하여 간격을 설정합니다.

```xml
<!-- Before (AvaloniaUI 스타일) -->
<StackPanel Spacing="40">
    <controls:OrangeFox41/>
    <controls:OrangeFox41/>
</StackPanel>

<!-- After (WPF 스타일) -->
<StackPanel>
    <controls:OrangeFox41 Margin="0,0,0,40"/>
    <controls:OrangeFox41/>
</StackPanel>
```

## 잠재적 런타임 에러 (직접 확인 필요)

### 1. Popup 위치 문제
- **증상**: 툴팁이 예상 위치에 표시되지 않을 수 있음
- **원인**: Popup의 Placement 및 Offset 계산이 CSS의 `left: 50%; transform: translate(-50%, -10px)`와 다를 수 있음
- **확인 필요**: 다양한 화면 해상도와 DPI 설정에서 테스트

### 2. 폰트 로딩 문제
- **증상**: Montserrat 폰트가 표시되지 않고 Segoe UI 폴백 사용
- **원인**: Montserrat 폰트가 시스템에 설치되어 있지 않은 경우
- **해결**: 필요시 폰트 파일을 리소스로 포함하거나 Google Fonts에서 설치

### 3. 애니메이션 타이밍
- **증상**: CSS transition과 WPF Storyboard 간의 이징 차이
- **원인**: CSS의 기본 transition과 WPF CubicEase 간의 동작 차이
- **확인 필요**: 시각적으로 비교 검증 필요
