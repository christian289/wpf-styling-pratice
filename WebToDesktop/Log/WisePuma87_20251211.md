## 컴파일 에러 및 수정 사항

### 에러 1: SplineEase 태그 없음

**에러 메시지:**
```
MC3074: XML 네임스페이스 'http://schemas.microsoft.com/winfx/2006/xaml/presentation'에 'SplineEase' 태그가 없습니다.
```

**원인:**
- WPF에는 `SplineEase`라는 EasingFunction이 존재하지 않음
- CSS의 `cubic-bezier(0.45, 0.05, 0.55, 0.95)`를 XAML로 변환 시 잘못된 클래스 사용

**수정 방법:**
- `DoubleAnimation` + `SplineEase` 대신 `DoubleAnimationUsingKeyFrames` + `SplineDoubleKeyFrame` 사용
- `KeySpline` 속성으로 cubic-bezier 값 직접 지정

```xml
<!-- Before (오류) -->
<DoubleAnimation Duration="0:0:0.5" To="0">
    <DoubleAnimation.EasingFunction>
        <SplineEase Spline="0.45,0.05,0.55,0.95" />
    </DoubleAnimation.EasingFunction>
</DoubleAnimation>

<!-- After (수정) -->
<DoubleAnimationUsingKeyFrames>
    <SplineDoubleKeyFrame KeyTime="0:0:0.5" Value="0" KeySpline="0.45,0.05,0.55,0.95" />
</DoubleAnimationUsingKeyFrames>
```

### 에러 2: SplineKeySpline 태그 없음

**에러 메시지:**
```
MC3074: XML 네임스페이스 'http://schemas.microsoft.com/winfx/2006/xaml/presentation'에 'SplineKeySpline' 태그가 없습니다.
```

**원인:**
- WPF에는 `SplineKeySpline`이라는 리소스 타입이 존재하지 않음
- `KeySpline`은 `SplineDoubleKeyFrame` 등에서 인라인으로만 사용 가능

**수정 방법:**
- `WisePuma87Resources.xaml`에서 해당 리소스 제거
- `KeySpline` 값은 `SplineDoubleKeyFrame`에서 인라인으로 사용

---

### 에러 3: StackPanel.Spacing 속성 없음

**에러 메시지:**
```
MC3072: XML 네임스페이스 'http://schemas.microsoft.com/winfx/2006/xaml/presentation'에 'Spacing' 속성이 없습니다.
```

**원인:**
- `StackPanel.Spacing`은 UWP/WinUI에서만 지원되는 속성
- WPF의 `StackPanel`에는 해당 속성이 없음

**수정 방법:**
- `Spacing="20"` 제거
- 각 자식 요소에 `Margin="0,0,0,20"` 추가

```xml
<!-- Before (오류) -->
<StackPanel Spacing="20">
    <controls:WisePuma87 />
</StackPanel>

<!-- After (수정) -->
<StackPanel>
    <controls:WisePuma87 Margin="0,0,0,20" />
</StackPanel>
```

## 잠재적 런타임 에러 (확인 필요)

### 1. StrokeDashOffset 애니메이션
- **우려 사항:** Path의 `StrokeDashOffset` 애니메이션이 예상대로 동작하지 않을 수 있음
- **원인:** `StrokeDashArray="24"`와 `StrokeDashOffset` 조합은 Path의 실제 길이와 연관됨
- **확인 필요:** 체크마크가 그려지는 애니메이션이 원본 CSS와 동일하게 보이는지 확인

### 2. 비대칭 border-radius 근사화
- **우려 사항:** CSS의 `border-radius: 30% 70% 70% 30% / 30% 30% 70% 70%` (blob 형태)를 WPF의 단일 `CornerRadius`로 완벽히 표현 불가
- **현재 구현:** `CornerRadius="10,6,10,6"` (근사값)
- **확인 필요:** 비대칭 모서리 효과가 원본과 얼마나 유사한지 확인

### 3. ColorAnimation의 Transparent 값
- **우려 사항:** `To="Transparent"` 애니메이션이 일부 케이스에서 예외 발생 가능
- **확인 필요:** 체크 해제 시 배경색 애니메이션 정상 동작 확인

## CSS → WPF 변환 매핑

| CSS 속성 | WPF 구현 |
|---------|---------|
| `transition` | `Storyboard` + `*Animation` |
| `stroke-dasharray` | `Path.StrokeDashArray` |
| `stroke-dashoffset` | `Path.StrokeDashOffset` |
| `cubic-bezier()` | `SplineDoubleKeyFrame.KeySpline` |
| `border-radius (비대칭)` | `Border.CornerRadius` (근사값) |
| `:checked` | `Trigger Property="IsChecked"` |
